# サービス管理システム Java実習課題一覧

本実習では、すでに作成されている「サービス管理システム（Webアプリケーション）」のソースコードを読み解き、仕様変更や機能追加を行うことで、Webアプリケーション開発の一連の流れを習得します。

## 必須課題

Webアプリケーションの基本的な仕組み（画面表示〜データベースアクセス）を自力で実装できるようになることを目的とした必須課題です。

各タスクごとに、以下を実施します。

- 設計書や実施することの理解
- コーディング
- 動作確認
- コードレビュー
- テストケースの作成・実施
- レビュー

### ① 単純な画面の修正

- **難易度:** ★☆☆☆☆☆
- **時間:** 7時間
- **概要:** 画面右上にログアウトボタンを表示し、ログアウトできるようにする。
- **実習内容:** 既存のHTMLテンプレート（Thymeleaf）の改修と、ログアウト処理を担うControllerの連携を理解し実装します。

### ② 単純な機能の修正

- **難易度:** ★★☆☆☆
- **時間:** 24時間
- **概要:** 加入者情報検索画面において、氏名で部分検索できるようにする。
- **実習内容:** 画面から入力された検索条件（氏名の一部）を受け取り、Spring JDBCを用いてデータベース（加入者情報テーブル）に対してLIKE句による検索処理を実行するよう修正します。

### ③ 検索画面の追加

- **難易度:** ★★☆☆☆
- **時間:** 27時間
- **概要:** 料金情報検索画面と機能を追加する。
- **実習内容:** 料金情報の検索処理や、検索結果を表示する画面をゼロから作成します。画面（HTML）からサーバー（Java/Spring）、データベース（SQL）までを貫通して実装する総合的なタスクです。

### 結合テスト

- **難易度:** ★★☆☆☆
- **時間:** 7時間
- **概要:** 実習で作成した機能を統合して、システムとして動作するか統合試験する。
- **実習内容:** 結合テスト仕様書を作成し、①〜③で実装したログアウト機能や検索機能が通しで正しく動作するかを確認し、エビデンス（スクリーンショット等）を作成します。

## 発展課題

必須タスクを早く完了した受講生に対して、現場でのより高度な実装力を養うために追加で提供する発展課題です。

### ④ 登録画面の追加

- **難易度:** ★★★☆☆
- **時間:** 31時間
- **概要:** 料金情報追加・編集画面と機能を追加する。
- **実習内容:** 料金情報の新規登録処理や編集処理、および料金情報編集画面を作成します。バリデーション（入力値チェック）や、データベースの更新・追加（INSERT/UPDATE）処理が含まれます。

### ⑤ バッチ処理の追加

- **難易度:** ★★★★☆
- **時間:** 46時間
- **概要:** 請求システムへ渡す月次請求情報の一括作成処理の追加。
- **実習内容:** 毎月1日に自動実行されるコマンドラインアプリケーションとして、加入者情報と料金情報を参照し、月額費用を集計して「請求データ」および「請求明細データ」を作成するバッチ処理を実装します。
