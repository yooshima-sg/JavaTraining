# サービス管理システム Java実習課題一覧

本実習では、すでに作成されている「サービス管理システム（Webアプリケーション）」のソースコードを読み解き、仕様変更や機能追加を行うことで、Webアプリケーション開発の一連の流れを習得します。

## 必須課題

Webアプリケーションの基本的な仕組み（画面表示〜データベースアクセス）を自力で実装できるようになることを目的とした必須課題です。

各タスクで、以下を実施します。

- 設計書や実施することの理解
- コーディング
- 動作確認
- コードレビュー
- テストケースの作成・実施
- 講師によるレビュー・承認

### ① 軽微な画面の修正

- **難易度:** ★☆☆☆☆
- **時間:** 7時間
- **概要:** 画面右上にログアウトボタンを表示し、ログアウトできるようにする。
- **実習内容:** 
    - 既存のHTMLテンプレート（Thymeleaf）を修正し、ログアウトボタンを実装する。
    - ログイン中の判定、ログアウト処理は、Spring Framework Security の機能を利用する。

### ② 軽微な機能の修正

- **難易度:** ★★☆☆☆
- **時間:** 24時間
- **概要:** 加入者情報検索画面において、氏名でも部分検索できるようにする。
- **実習内容:** 
    - 加入者検索画面に氏名の一部の入力を受け付ける、入力項目を実装する。
    - 入力された検索条件を受け取り、Spring JDBCを用いて加入者情報テーブルに対してLIKE句による検索処理を実行するよう修正します。

### ③ 検索画面の追加

- **難易度:** ★★★☆☆
- **時間:** 27時間
- **概要:** 料金情報検索画面と機能を追加する。
- **実習内容:**
    - 設計書を読み、料金情報の検索処理や、検索結果を表示する画面をゼロから作成します。
    - 画面（HTML）からサーバー（Java/Spring）、データベース（SQL）までを貫通して実装します。
    - データはデータベースに直接アクセスして投入します。

### 結合テスト

- **難易度:** ★★☆☆☆
- **時間:** 7時間
- **概要:** 実習で作成した機能を統合して、システムとして動作するか統合試験する。
- **実習内容:**
    - ①〜③で実装したログアウト機能や検索機能が通しで正しく動作するかを確認する結合テスト仕様書を作成をします。
    - 作成した結合テスト仕様書に沿って、動作確認及びエビデンス（スクリーンショット等）を作成します。

## 発展課題

必須タスクを早く完了した受講生に対して、現場でのより高度な実装力を養うために追加で提供する発展課題です。

### ④ 登録画面の追加

- **難易度:** ★★★☆☆
- **時間:** 31時間
- **概要:** 料金情報追加・編集画面と機能を追加する。
- **実習内容:** 
    - 料金情報の新規登録処理や編集処理、および料金情報編集画面を作成します。
    - バリデーション（入力値チェック）や、データベースの更新・追加（INSERT/UPDATE）処理を行います。

### ⑤ バッチ処理の追加

- **難易度:** ★★★★☆
- **時間:** 46時間
- **概要:** 請求システムへ渡す月次請求情報の一括作成処理の追加。
- **実習内容:** 
    - 毎月1日に自動実行されるコマンドラインアプリケーションとして作成します。
    - 加入者情報と料金情報を参照し、月額費用を集計して「請求データ状況」を管理しながら「請求データ」および「請求明細データ」を作成するバッチ処理を実装します。
